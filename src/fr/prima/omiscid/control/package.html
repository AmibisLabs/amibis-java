This package contains the classes use to create a service according to the definition given in the Basic Interconnexion Protocol (BIP). A service BIP implies a registration to DNS-SD, and the creation of a TCP service for the service control. This package contains also classes to discover service, and to do queries to the control port.

The registration to DNS-SD enables to discover this service. The service are registered to DNS-SD with the type "_bip._tcp". The control port enables to obtain a description of the service through the list of its inputs, its outputs and its variables.

How to create a service ?

To create a service, the user must extend one of these two classes:

  • ControlServer
  • ServiceFromXml
The second creates a ControlServer and then load a XML description that contains data about variables, input and output. These data are for example name, description.

The methods that can need to be reimplemented are modifVariable and connect called to answer queries about a modification of a variable or a connection to do.

The item of a service : variable, input, output are manipulated through the classes VariableAttribut, and InOutputAttribut;

How to find a service ?

The class BrowseForService enables to list the service registered on DNS-SD with a given type.

The class WaitForBipService enables to find a particular service registered on DNS-SD with the type "_bip._tcp".

How to question a service ?

To interact with the control port, there is the class ControlClient. It enables to send query, then it waits for answer, and parses it. With BIP, the peer mus be identified by an identifier, it can be obtained by using the static method BipService.GenerateServiceId.

Note : A good way to obtain a ControlClient instance for a service is to used the class BipService, and call the method initControlClient.
